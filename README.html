About the Project¶

This is the documentation for the OSU HART Launch Controller, the user interface side of the Launch System.

Built with¶

  • Sphinx documentation generator

  • TexLive typesetting system for formatting the PDF output

Getting Started¶

The documentation is built and deployed automatically using a GitHub Actions workflow. If you want to build the documentation locally, the easiest way is to use the OSU HART Project Workspace, which comes with all the necessary dependencies pre-installed. Optionally, you can install the dependencies yourself with the following commands.

Prerequisites¶

  • Python PIP

    $ sudo apt-get install python3-pip

  • Sphinx documentation generator

    $ pip3 install -U sphinx

    Make sure ~/.local/bin is added to PATH

  • GNU Make

    $ sudo apt-get install build-essential

  • recommonmark

    $ pip3 install recommonmark sphinx-markdown-tables

If you plan on building LaTeX PDF documentation, you’ll also need the following packages:

$ sudo apt-get install texlive-latex-recommended texlive-fonts-recommended texlive-latex-extra latexmk

Installing¶

  1. Clone the official repository

    $ git clone https://github.com/HART-Avionics/OSU-HART-User-Interface.git

Usage¶

The manual documentation is written mostly in Markdown, but Sphinx also has it’s own default markup format called reStructuredText which can also be used for generating documentation from comments in source code.

The Sphinx configuration is set in docs/conf.py. For the full list of available options, see the official documentation.

Once you’re finished making changes, you can build the documentation.

$ cd docs
$ make build

Sphinx will generate both HTML & LaTeX PDF documentation in the docs/build/ directory.

You can build the HTML and LaTeX PDF documentation individually with

$ make html

or

$ make latexpdf LATEXMKOPTS="-silent"

For more examples, please refer to the official Sphinx documentation
